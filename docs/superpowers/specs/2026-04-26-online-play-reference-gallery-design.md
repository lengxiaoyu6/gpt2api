# OnlinePlay 图生图参考图区卡片化设计

**目标**

将 `web` 端图生图参考图区从“主预览画布 + 缩略条”调整为紧凑的方形卡片集合，减少纵向占用，并保持多图上传、删除、预览与继续编辑的既有能力。

## 现状

当前 `web/src/views/personal/OnlinePlay.vue` 的参考图区包含一个高度较大的 `compare-canvas--reference` 主画布，以及一个横向 `thumb-strip` 缩略条。该结构在桌面端和移动端都会占用较多高度，尤其在参数表单位于同一列时，首屏信息密度偏低。

现有状态以 `activeRefIndex` 和 `activeRefImage` 为核心，交互方式是点击缩略图切换主图。图片预览能力已经存在于 `openPreview` 与 `el-image-viewer`，结果图区也已经使用卡片式网格。

## 设计方案

### 布局结构

参考图区改为单一的参考图卡片网格，移除以下两部分：

`compare-canvas compare-canvas--reference`

`thumb-strip`

新结构使用一个统一的 `ref-card-grid` 容器承载：

1. 空状态上传卡片
2. 已上传参考图卡片列表
3. “添加图片”卡片

结果图区保持现状，仅修改参考图区。

### 卡片行为

每张参考图以 `1:1` 方形卡片展示，图片采用 `object-fit: cover`，以保证小尺寸浏览时稳定、整齐。

卡片交互如下：

1. 点击图片卡片，调用现有 `openPreview`，将 `refImages` 全量数据 URL 作为预览列表，并以当前索引作为初始项。
2. 删除按钮悬浮在卡片右上角，点击后仅删除当前卡片，不触发预览。
3. “添加图片”入口作为与图片卡片同尺寸的上传卡片并入网格。
4. 空状态下上传卡片单独显示；已有图片后上传卡片始终作为最后一个网格项显示。

### 文案与信息层

参考图区标题、副文案和张数统计保留。

每个图片卡片底部保留一层简洁的信息条，展示文件名；大小信息以更弱化的文本呈现，避免额外占据高度。主画布模式中的大面积文件信息浮层不再保留。

### 状态与逻辑调整

由于参考图区不再存在“主图切换”概念，可移除以下状态与方法：

`activeRefIndex`

`activeRefImage`

`setActiveRef`

保留并复用以下逻辑：

1. `refImages`
2. `handleFilePick`
3. `removeRefImage`
4. `continueEditCurrentResult`
5. `openPreview`

`continueEditCurrentResult` 仍然将当前结果图写回 `refImages`，但无需再重置参考主图索引。

### 响应式策略

参考图区卡片网格采用自适应列布局，建议使用 `repeat(auto-fill, minmax(104px, 1fr))` 或同等级写法。

桌面端会呈现更紧凑的多列排列；移动端自动收缩为双列或三列，保证触控面积。所有卡片保持正方形，整体高度随图片数量自然增长，不再预留一块固定大画布高度。

### 测试调整

更新 `web/tests/online-play-img2img-layout.node.test.mjs`，同步反映新的参考图区实现：

1. 删除对 `thumb-strip`、`activeRefIndex`、`activeRefImage`、`setActiveRef` 的依赖断言。
2. 新增对 `ref-card-grid`、参考图卡片、上传卡片、点击预览行为的断言。
3. 将“大尺寸参考图固定画布约束”断言改为“参考图卡片网格与方形卡片约束”断言。
4. 保留结果图区现有主图区与操作区断言，避免扩大修改范围。

## 错误处理

上传大小限制、空参考图提交限制、空提示词限制与上游请求异常处理保持现状。当前设计仅调整参考图区展示方式，不修改鉴权、请求协议与价格计算逻辑。

## 验收标准

满足以下条件视为完成：

1. 参考图区首屏不再出现大尺寸主画布。
2. 每张参考图以方形卡片形式直接显示。
3. 点击任意参考图卡片可以打开现有全屏预览器。
4. 删除参考图、追加参考图、继续编辑当前结果均保持可用。
5. 移动端参考图区高度明显降低，且卡片保持足够触控面积。
6. 相关静态测试通过。
