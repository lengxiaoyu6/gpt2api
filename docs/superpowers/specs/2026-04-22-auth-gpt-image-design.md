# 认证页改版设计文档

## 目标

将现有登录页与注册页从“管理后台入口”调整为“GPT-image 中转 API 平台认证入口”，使首屏在短时间内传达以下信息：

1. 平台主营 GPT-image 图像生成接口接入。
2. 支持文生图、图生图、批量生成。
3. 登录后进入控制台管理密钥、额度、任务与调用记录。
4. 注册后可领取体验额度并尽快开始调用。

目标受众为开发者与零散客户，首要动作为登录控制台。

## 当前问题

现有 `web/src/views/auth/Login.vue` 左侧内容偏向多账号池、权限、审计、积分结算，认证页语义更像后台系统入口。`web/src/views/auth/Register.vue` 仅有表单卡片，缺少体验额度、图像能力与平台识别信息。移动端会完全隐藏品牌说明区，导致平台身份在窄屏上消失。

## 设计方案

认证页统一为双栏结构：左侧为品牌能力区，右侧为认证表单区。登录页与注册页共用外壳，仅替换右侧表单内容。

### 品牌能力区

品牌能力区承担“平台是什么”与“支持什么能力”的职责，信息顺序固定为：

1. 站点标识与平台标签。
2. 主标题“统一接入 GPT-image 能力”。
3. 副标题，说明面向开发调试、个人项目与小规模业务接入。
4. 三张能力卡片：文生图、图生图、批量生成。
5. 接口规格块，展示 `POST /v1/images/generations` 与关键字段。
6. 底部服务说明，展示 OpenAI Images 兼容、控制台能力、体验额度信息。

移动端保留压缩版品牌能力区，不再整块隐藏。

### 登录页表单区

右侧标题为“登录控制台”，副标题说明登录后可管理 API 密钥、图像任务、调用记录与账户额度。表单仍为邮箱与密码，按钮文案为“登录控制台”。

表单区提供以下辅助信息：

1. 登录失败时在卡片内展示错误信息。
2. 开放注册时展示“首次使用可先注册账号并领取体验额度”与注册链接。
3. 关闭注册时展示邀请开通方式说明。

### 注册页表单区

右侧标题为“注册账号”，副标题为“领取体验额度，开始接入 GPT-image API”。表单顶部加入绿色提示条，说明新账号赠送体验额度，注册后即可进入控制台创建 API 密钥。主按钮文案为“注册并进入控制台”。

关闭注册时，提示条替换为邀请开通说明，页面保持表单结构但整体禁用。

## 视觉系统

整体采用深色开发工具风格：

1. 页面主背景使用深蓝灰渐变与轻量径向光效。
2. 面板为半透明深色卡片，辅以细边框与柔和阴影。
3. 主强调色为蓝色，图像能力采用蓝紫与青色点缀。
4. 参数、标签、接口字段使用等宽字体风格。
5. 保持信息密度高但分层清晰，避免营销页式大面积视觉噪音。

## 组件边界

为避免登录页与注册页继续各自堆积样式，拆分为以下组件：

1. `web/src/components/auth/AuthShell.vue`
   负责整页背景、双栏布局、移动端堆叠、站点页脚信息承载。
2. `web/src/components/auth/AuthHeroPanel.vue`
   负责品牌标签、标题、能力卡片、接口规格块与服务说明。
3. `web/src/components/auth/AuthFormCard.vue`
   负责统一的表单卡片框架、标题、副标题、提示条与底部说明。
4. `web/src/views/auth/Login.vue`
   保留登录字段、校验与跳转逻辑。
5. `web/src/views/auth/Register.vue`
   保留注册字段、校验、自动登录与跳转逻辑。

## 数据来源

继续使用 `useSiteStore` 作为认证页公开站点信息来源：

1. `site.name`
2. `site.description`
3. `site.logo_url`
4. `site.footer`
5. `auth.allow_register`

其中 Logo、站点名与页脚沿用站点配置；图像能力卡片、接口规格块与体验额度提示作为认证页固定内容。默认站点描述应调整为图像 API 平台语义。

## 测试与验证

本次改版至少覆盖以下验证内容：

1. 静态测试验证登录页与注册页改为复用认证外壳组件。
2. 静态测试验证品牌能力区包含 GPT-image 能力说明与接口规格块。
3. 静态测试验证登录页、注册页关键文案与按钮文案更新。
4. 构建验证确认 Vue 类型检查与打包通过。
5. 桌面与移动宽度下，品牌能力区均可见且结构完整。
