# WAP 端积分使用记录局部详情页设计文档

## 目标

在 `wap` 端“我的”页面内新增一个局部详情页入口。点击“可用积分”统计卡片后，页面切换到“积分使用记录”视图，并在视图顶部提供返回按钮，返回到个人中心主页。

积分使用记录页展示当前账号的全部积分流水，而非仅消费记录。页面继续复用现有移动端样式体系，不调整底部导航与全局壳层结构。

## 范围

本次修改限定在 `wap` 目录内，包含以下内容：

1. 个人中心页中“可用积分”卡片增加点击行为。
2. `Profile` 组件内部新增“主页”与“积分使用记录页”两种局部视图状态。
3. 新增积分流水列表请求封装与展示组件。
4. 在列表中展示全部积分流水的类型、变动值、余额、备注、时间。
5. 补充对应前端测试。

以下内容保持现状：

1. 底部导航结构保持现状。
2. 全局路由与标签页状态保持现状。
3. 充值、签到、会员弹窗逻辑保持现状。
4. 后端接口与 `web` 端页面保持现状，仅复用已有接口契约。

## 现状

当前 `wap/src/components/views/Profile.tsx` 已经展示“今日奖励”和“可用积分”两个统计卡片，但卡片仅作静态展示，没有跳转行为。

项目后端已经提供个人积分流水接口 `GET /api/me/credit-logs`。`web/src/api/me.ts` 已定义 `MyCreditLog` 与 `listMyCreditLogs`，`web/src/views/personal/Usage.vue` 已展示积分流水列表，可作为字段与文案参考。

`wap` 端目前尚未封装该接口，也没有积分流水页面或局部详情页状态。

## 方案比较

### 方案一

点击“可用积分”后切换到底部导航“记录”页，并在原有图片任务记录页中混合展示积分流水。

优点是页面数量少。缺点是“图片任务历史”和“积分流水”属于两类不同信息，混合后页面语义会变得模糊，移动端浏览成本较高。

### 方案二

在 `Profile` 组件内部增加局部详情态。点击“可用积分”后，个人中心切换到积分流水子页；点击返回按钮后，再回到个人中心主页。

优点是页面边界清楚，改动集中，保留当前底部导航结构，符合移动端二级详情页的常见交互方式。

### 方案三

使用弹窗或抽屉展示积分流水列表。

优点是实现较快。缺点是长列表浏览和后续扩展分页能力时体验一般，页面空间也偏紧。

当前采用方案二。

## 总体设计

个人中心页内部增加一个局部视图状态，例如 `profile` 与 `creditLogs`。默认进入 `profile` 视图，点击“可用积分”卡片后切换为 `creditLogs` 视图。

积分流水页由三部分组成：

1. 顶部栏：返回按钮与标题“积分使用记录”。
2. 余额摘要：展示当前可用积分，作为页面的简要信息头部。
3. 列表区域：展示全部积分流水，按时间倒序排列。

返回按钮只影响 `Profile` 内部状态，不修改全局 `activeTab`。

## 数据接口

`wap` 新增 `wap/src/api/credit.ts`，封装以下类型与请求：

```ts
export interface CreditLogItem {
  id: number
  user_id: number
  key_id: number
  type: string
  amount: number
  balance_after: number
  ref_id: string
  remark: string
  created_at: string
}

export function listMyCreditLogs(params?: { limit?: number; offset?: number })
```

请求地址为：

`GET /api/me/credit-logs`

首版采用分页参数 `limit` 与 `offset`，与后端现有接口保持一致。首次进入积分流水页时拉取第一页数据。

## 组件划分

### `Profile.tsx`

负责：

1. 维护局部视图状态。
2. 在主页中为“可用积分”卡片绑定点击事件。
3. 在详情页与主页之间切换。
4. 继续保留签到、充值、退出登录等既有行为。

### `ProfileCreditLogs.tsx`

新增个人中心积分流水子组件，负责：

1. 在挂载时或首次显示时请求积分流水。
2. 渲染返回栏、余额摘要、流水列表。
3. 处理加载中、空列表与普通列表三种状态。

组件通过属性接收：

1. `balance` 当前积分余额。
2. `onBack` 返回事件。

## 交互细节

### 进入与返回

点击“可用积分”统计卡片后，切换到积分流水页。

点击顶部返回按钮后，回到个人中心主页。返回后原有页面数据保持现状，无需重新拉取签到、历史与用户信息。

### 列表内容

每条流水展示：

1. 类型名称。
2. 积分变动值，增加显示为正值样式，扣减显示为负值样式。
3. 变动后余额。
4. 备注。备注为空时显示简短占位文案。
5. 创建时间。

类型名称可在前端做静态映射，参考 `web/src/views/personal/Usage.vue` 中的映射：

1. `recharge` 显示“充值”。
2. `consume` 显示“消费”。
3. `refund` 显示“退款”。
4. `admin_adjust` 显示“调账”。
5. `redeem` 显示“兑换码”。
6. `checkin` 显示“签到”。
7. `freeze` 显示“冻结”。
8. `unfreeze` 显示“解冻”。

### 加载状态

首次进入页面时显示加载提示。

接口成功但无数据时显示空态文案。

接口失败时显示错误提示，并保留返回按钮与页面标题。首版采用提示组件反馈错误，并在页面中展示空态，不增加重试按钮。

## 数据流

1. `Profile` 默认展示主页。
2. 用户点击“可用积分”卡片。
3. `Profile` 将局部视图状态切换为 `creditLogs`。
4. `ProfileCreditLogs` 发起 `listMyCreditLogs({ limit: 20, offset: 0 })` 请求。
5. 请求成功后渲染列表；请求失败时弹出提示并展示空态。
6. 用户点击返回按钮后，局部视图状态切回 `profile`。
