# 后台登录页收敛设计

## 目标

将 `web` 端登录页收敛为常规后台登录页，页面视觉回到单栏居中表单，移除偏展示页的能力介绍区，同时保留现有管理员身份校验、登录错误提示与后台首页跳转逻辑。

## 范围

本次调整只涉及 `web` 管理后台登录入口。

包含内容：

1. `Login.vue` 改为单栏居中后台登录卡片。
2. 登录页移除 `AuthHeroPanel` 引用与图像能力展示文案。
3. `AuthShell.vue` 增加无英雄区时的单栏布局支持。
4. 测试断言改为验证常规后台登录页结构与现有登录逻辑。

不包含内容：

1. 后端认证接口调整。
2. 管理员权限校验逻辑调整。
3. `wap` 端任何页面修改。

## 页面结构

登录页保留 `AuthShell` 与 `AuthFormCard` 两个共享组件，但取消左侧英雄区插槽。页面主体改为居中的后台登录卡片，卡片内部自上而下展示：

1. 系统名称与简短标识。
2. 标题“后台登录”。
3. 简洁说明文字，强调仅限管理员账号。
4. 邮箱与密码表单。
5. 登录失败提示。
6. 登录按钮。
7. 卡片底部的后台说明与返回信息。

## 视觉原则

采用常规后台登录页风格：

1. 维持深色背景与现有品牌色，减少大面积内容展示。
2. 表单卡片居中，宽度控制在常见后台登录框范围。
3. 说明文案压缩为后台用途说明，移除产品营销语义。
4. 保留清晰聚焦态、错误提示与移动端适配。

## 测试策略

先改 `web/tests/auth-pages.node.test.mjs`，新增或调整断言以覆盖：

1. 登录页继续使用 `AuthShell` 与 `AuthFormCard`。
2. 登录页源码中已移除 `AuthHeroPanel`。
3. 登录页包含“后台登录”等常规后台文案。
4. 登录成功后的管理员校验与 `/admin/dashboard` 跳转逻辑保持不变。

随后运行单测，先看到失败，再修改实现，再执行完整测试与构建验证。
