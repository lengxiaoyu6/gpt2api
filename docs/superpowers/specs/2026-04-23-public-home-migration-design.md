# 公开首页迁移设计

## 目标

新增公开营销首页与配套公开页面，路由使用 `/`、`/showcase`、`/pricing`、`/login`、`/register`，视觉结构尽量贴近 `nexusai---gpt-image-api-中转平台` 目录中的参考实现，同时保留当前项目的 Vue 路由、登录注册接口与控制台页面。

## 设计

公开页面使用独立营销布局，与登录后控制台布局分离。营销布局负责顶部导航、暗色背景、页脚与统一容器；首页、案例页、定价页、登录页、注册页仅负责各自内容区。

首页保留参考实现中的五个核心区域：顶部导航、Hero 区、三列能力卡片、案例预览区、底部行动召唤区。按钮跳转改为 Vue Router 导航，进入登录、注册、案例页、定价页与控制台页面。

案例页迁移参考实现中的图片瀑布式展示与筛选标签，先使用静态示例数据，不新增后台接口。

定价页迁移参考实现中的价格表与附加说明，按钮接入当前项目的注册或充值入口。

登录页与注册页沿用当前认证接口与用户状态管理，但展示样式改为参考实现的深色卡片表单。登录成功后继续跳转 `/personal/dashboard` 或 `redirect` 指定页面，注册成功后自动登录并进入控制台。

## 测试范围

新增 Web 静态测试验证：

1. `/` 不再重定向到 `/personal/dashboard`。
2. 路由表声明 `/`、`/showcase`、`/pricing` 三个公开营销页面。
3. 首页包含 Hero、能力卡片、案例预览、行动召唤区。
4. 登录与注册页包含新的营销式文案与返回首页入口。
5. 案例页与定价页存在对应静态内容。
