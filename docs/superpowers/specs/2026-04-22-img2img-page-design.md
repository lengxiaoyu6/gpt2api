# 图生图页面交互整理设计

## 目标

整理 `web/src/views/personal/OnlinePlay.vue` 中图生图区的上传区与结果区交互，使页面围绕“参考图与生成结果并排对照”展开，突出查看、放大、下载三项主要操作，并将继续编辑降为次级入口。

本次调整以现有图生图功能已经可用为前提，移除过时的 Preview 占位提示，补齐空态、生成中、生成完成三种状态下的稳定体验。

## 现状

当前图生图区仍保留一条过时的警告提示，文案表明图生图协议尚未接入、提交会返回 `501`。这一信息与当前实际可用状态冲突。

左侧区域目前以表单堆叠为主，参考图上传完成后仍然停留在“上传框 + 缩略网格”的结构，没有形成稳定的主参考图焦点。

右侧结果区当前沿用与文生图接近的宫格展示方式。查看与下载操作附着在缩略图悬浮层中，继续编辑入口尚未建立，结果区也没有围绕“对照”来组织。

图生图请求当前继续通过 `playGenerateImage` 发送，参数中包含 `reference_images`。本次设计聚焦页面结构与交互层次，保持这一调用链路不变。

## 目标行为

### 页面主结构

图生图区采用左右双栏布局。

左侧为参考图区，承载参考图上传、参考主图查看、参考图切换、改动描述与输出比例。

右侧为结果区，承载结果主图、结果切换、状态反馈与结果操作。页面从进入图生图标签起即展示这一双栏骨架，避免状态切换时整体跳动。

### 参考图区

参考图区采用“主预览 + 缩略条”结构。

当尚未上传参考图时，主预览区域显示上传引导，支持点击选择文件，原有拖拽文案保留在该区域内。

当已上传参考图后，主预览区域显示“当前参考主图”。若存在多张参考图，主预览下方展示横向缩略条，用于切换当前参考主图；缩略条末尾保留“添加图片”入口。

删除操作保留在缩略图层级，而非主预览中央，避免误触。切换缩略图时仅更新当前参考主图状态，不影响提示词、比例与已生成结果。

### 结果区

结果区采用“主结果图 + 缩略条”结构。

右侧主体始终保留固定画布区域。空态时显示结果占位说明；生成中时在同一画布区域显示进度反馈；生成完成后在原位置替换为结果主图。

如果当前返回多张结果，结果主图下方展示横向缩略条，用于切换当前结果主图。若仅有一张结果，则缩略条隐藏。

这种结构与左侧参考主图形成稳定的一对一并排关系，便于在修改提示词或切换参考图时快速对照。

### 操作层级

结果区主图下方固定展示三项主要操作：

`查看`：在新窗口打开当前结果原图。  
`放大`：调用现有全屏图片预览器，并定位到当前结果图。  
`下载`：沿用现有下载逻辑下载当前结果图。

`继续编辑当前结果` 作为次级入口，以弱化文字按钮或轻样式链接呈现，位置放在主操作带右侧或结果说明区右下侧，避免与三项主要操作形成同级竞争。

### 继续编辑语义

点击 `继续编辑当前结果` 后，当前结果图写回左侧参考图区，并立即成为新的参考主图。

原有参考图列表清空，避免把“继续编辑当前结果”误解为在旧参考图集合上追加一张结果图。提示词内容保留，方便围绕当前结果继续细化；输出比例保持不变；结果区保留当前结果，直到新的生成请求发出。

### 状态切换

右侧结果区状态分为四种：

空态展示占位插画或占位块，并提示“先在左侧上传参考图并填写改动描述”。

生成中展示加载反馈、进度文案与保持页面开启的提醒，但仍占据同一结果画布。

生成完成展示结果主图、预览模式提示与主操作带。若接口返回 `is_preview`，则在结果画布上方保留轻量提示条，沿用现有降级语义。

错误态沿用现有错误块，但错误块位于结果画布内部，保持整体布局稳定。

## 实现方式

### 脚本状态

在 `OnlinePlay.vue` 的图生图部分新增两个显式状态：

`activeRefIndex` 用于表示当前参考主图序号。  
`activeResultIndex` 用于表示当前结果主图序号。

参考图上传、删除、继续编辑当前结果后，都需要维护 `activeRefIndex` 的回退与指向规则。生成成功后，`activeResultIndex` 默认回到首张结果；切换结果缩略图时只更新该索引。

当前请求协议保持 `playGenerateImage` 与 `reference_images` 方案，避免本次页面整理扩展为上传协议切换。

### 模板调整

图生图区模板从“左侧表单 + 右侧宫格”调整为“左侧参考面板 + 右侧结果面板”。

需要完成的模板变更包括：

移除图生图区顶部过时的 Preview 警告提示。  
将左侧上传框改造成可承载空态与主参考图的统一预览区域。  
在左侧新增参考图缩略条与添加图片入口。  
在右侧新增固定结果画布、结果缩略条与主操作带。  
将现有悬浮图标操作改为下方显式动作按钮。  
新增弱化样式的 `继续编辑当前结果` 入口。

### 样式调整

样式需要围绕以下目标整理：

左右双栏在桌面宽度下保持平衡比例，参考图与结果图拥有接近的主画布高度。  
主画布使用统一圆角、边框与浅背景，形成对照感。  
缩略条采用低干扰样式，当前项通过边框或底色高亮。  
主操作带使用显式按钮组，次级入口使用弱化文字样式。  
空态、生成中、错误态复用同一画布容器，减少高度跳动。

## 测试范围

新增或调整 `web/tests` 下针对在线体验页的静态测试，覆盖以下断言：

`OnlinePlay.vue` 不再包含“图生图目前处于 Preview”与“当前提交会返回 501”这类过时文案。  
图生图区存在参考主图与结果主图的结构标识。  
页面存在 `activeRefIndex` 与 `activeResultIndex` 或等价状态。  
结果区存在查看、放大、下载三项主要操作，以及 `继续编辑当前结果` 次级入口。  
继续编辑逻辑会把当前结果图回写到参考图区。  
结果区采用固定画布承接空态、生成中、完成态，而非仅保留结果宫格。

## 非目标

本次调整不修改后端接口，不改动模型选择、计费规则、预览模式判定，也不把当前图生图请求从 `playGenerateImage` 切换到 `playEditImage`。

本次调整也不重做文生图区，仅在必要范围内复用已有图片预览与下载能力。
